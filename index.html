세 가지 프로토콜, 하나의 시스템

각 프로토콜이 담당하는 역할과 그 작동 방식을 확인하세요.

A2A

Agent-to-Agent Protocol

에이전트 간 실시간 협업의 핵심 레이어. 전략, 베팅, 모니터링, 미디어 에이전트가 실시간으로 소통합니다.

전략 ↔ 베팅
베팅 ↔ 모니터링
전략 ↔ 미디어
성과검증 ↔ 전체
전략 에이전트 → (A2A) → 베팅 에이전트 풀:
  마켓 ID: POL-2024-BILL-7734
  목표 오즈: 42% → 68%
  예산: $80,000
  분산 계좌 수: 340개
핵심 결제 레이어

x402

HTTP 402 Micropayments

수백 개 에이전트가 수천 번 소액 베팅을 자동 실행. API 호출 단위의 마이크로결제로 에이전트 자율 실행의 핵심.

베팅 자금 자율 분배
성과 기반 자동 정산
에스크로 자동 해제
베팅 에이전트 #237:
  POST /polymarket/bet
  x-payment: $47.50 (x402 자동 처리)
  body: { market: "POL-7734",
          position: "YES",
          amount: 47.50 }
  → 즉시 실행, 영수증 자동 발행

AP2

Agent Payment Protocol v2

신뢰와 감사의 레이어. 성과 공증, 에스크로 계약, 내부 감사 로그를 불변 기록으로 관리합니다.

① 성과 공증
② 에스크로 계약
③ 내부 감사 로그

x402 Deep Dive

왜 x402인가?

HTTP 402 "Payment Required" 상태 코드를 활용한 오픈 결제 프로토콜

마이크로페이먼트

$0.001부터 가능한 초소액 결제. Base 체인의 낮은 가스비로 실현 가능해졌습니다.

계정 불필요

회원가입, 로그인, API 키 없이 암호화 서명만으로 결제가 완료됩니다.

HTTP 네이티브

기존 HTTP 인프라 위에서 작동. 특별한 SDK 없이도 구현 가능합니다.

즉시 정산

신용카드의 며칠 정산과 달리, ~2초 만에 온체인 최종 확정됩니다.

AI 에이전트 최적화

AI가 자율적으로 결제할 수 있는 구조. MCP, Agent-to-Agent 결제 지원.

오픈 프로토콜

Coinbase가 주도하고, 다양한 기업이 지원하는 개방형 표준입니다.

핵심 구성 요소

클라이언트

결제 요청을 보내고 EIP-712 서명을 생성하는 주체. AI 에이전트 또는 브라우저.

리소스 서버

유료 콘텐츠를 제공하는 서버. 402 응답으로 결제 정보를 전달.

퍼실리테이터

서명을 검증하고 온체인 정산을 처리하는 중개자. Coinbase 등이 운영.
